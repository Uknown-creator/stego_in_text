# Стеганография в тексте

Система скрытой передачи сообщений: открытый текст шифруется симметрично, ключ
согласуется по протоколу **Диффи-Хеллмана**, а шифртекст представляется в виде
**слов русского языка** и пересылается между двумя клиентами через сервер по
**WebSocket**. Перехватчик видит лишь безобидный набор слов.

## Назначение и принцип работы

1. Каждый клиент имеет долгоживущую пару ключей Диффи-Хеллмана (хранится в файле
   ключа). Публичный ключ публикуется на сервере под идентификатором клиента.
2. Отправитель запрашивает публичный ключ получателя, вычисляет общий секрет DH,
   выводит из него 256-битный ключ AES (SHA-256) и шифрует файл алгоритмом
   **AES-256-GCM** (библиотека Crypto++).
3. Байты шифртекста кодируются словами из словаря (256 слов, по слову на байт) —
   получается «текст».
4. «Текст» отправляется на сервер, который складывает его в очередь получателя
   (модель store-and-forward — получателю не обязательно быть онлайн).
5. Получатель забирает сообщение, декодирует слова обратно в байты, вычисляет тот
   же общий секрет DH и расшифровывает сообщение.

## Состав проекта

| Компонент | Каталог | Назначение |
|-----------|---------|------------|
| Клиент    | `client/` | криптография (DH+AES), стеганография, WebSocket-клиент, CLI |
| Сервер    | `server/` | WebSocket-ретранслятор: реестр ключей и очереди сообщений |
| Словарь   | `data/dictionary.txt` | 256 уникальных русских слов (UTF-8) |

## Зависимости

Подтягиваются автоматически через **vcpkg** (manifest-режим, `vcpkg.json`):
`cryptopp`, `ixwebsocket`, `nlohmann-json`, `doctest`.

## Установка и сборка

Требуется CMake ≥ 3.21, компилятор C++17 и vcpkg.

```bash
export VCPKG_ROOT=/путь/к/vcpkg          # vcpkg должен быть склонирован и забутстрапен
cmake --preset default                   # установит зависимости и сконфигурирует
cmake --build build                      # соберёт клиент и тесты
```

## Запуск тестов

```bash
ctest --test-dir build --output-on-failure
```

## Сценарии использования

Запустить сервер (по умолчанию порт 8080):

```bash
./build/server/server --port 8765
```

Получатель `bob` должен один раз зарегистрироваться, чтобы его публичный ключ
оказался на сервере:

```bash
./build/client/client --mode receive \
  --keyfile bob.key --dictionary data/dictionary.txt \
  --server ws://localhost:8765 --id bob
```

Отправитель `alice` шифрует файл и отправляет его `bob`:

```bash
./build/client/client --mode send \
  --plaintext message.txt --keyfile alice.key --dictionary data/dictionary.txt \
  --server ws://localhost:8765 --id alice --to bob
```

`bob` забирает сообщение (с выводом в файл):

```bash
./build/client/client --mode receive \
  --keyfile bob.key --dictionary data/dictionary.txt \
  --server ws://localhost:8765 --id bob --out received.txt
```

### Параметры клиента

| Параметр       | Назначение                                   | Режим |
|----------------|----------------------------------------------|-------|
| `--mode`       | `send` или `receive`                         | оба   |
| `--keyfile`    | файл с приватным ключом DH (создаётся при первом запуске) | оба |
| `--dictionary` | путь к словарю слов                          | оба   |
| `--server`     | адрес WebSocket-сервера                      | оба   |
| `--id`         | идентификатор текущего клиента               | оба   |
| `--to`         | идентификатор получателя                     | send  |
| `--plaintext`  | путь к файлу с открытым текстом              | send  |
| `--out`        | файл для сохранения принятого текста (иначе вывод в консоль) | receive |

Все пути задаются пользователем, абсолютные пути в коде отсутствуют, разделитель — `/`.

## Безопасность

AES-256-GCM обеспечивает не только конфиденциальность, но и проверку целостности
(тег аутентификации): подмена или повреждение шифртекста приводит к ошибке
расшифрования. Для каждого сообщения генерируется случайный вектор инициализации.
